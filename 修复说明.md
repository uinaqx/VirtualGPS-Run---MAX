# 虚拟跑步 - Bug 修复说明

## 修复内容

### 1. 地图显示问题（白屏）
**问题原因**：osmdroid 使用旧的 tile 加载机制，在国内网络环境下不稳定

**修复方案**：
- 将地图引擎从 osmdroid 更换为 **MapLibre**
- MapLibre 使用矢量地图渲染，性能更好，加载更稳定
- 使用 CartoDB Voyager 样式（国内可访问）

### 2. 虚拟定位不生效
**问题原因**：
- 仅模拟了 GPS_PROVIDER，但现代应用通常使用 FUSED_PROVIDER
- Location 对象缺少必要的字段（如 elapsedRealtimeNanos）

**修复方案**：
- 同时模拟三个定位 Provider：
  - `GPS_PROVIDER` - GPS 卫星定位
  - `NETWORK_PROVIDER` - 网络定位
  - `FUSED_PROVIDER` - 融合定位（Android 最新推荐）
- 补充完整的 Location 字段：
  - `elapsedRealtimeNanos` - 系统启动时间（必需）
  - `speed` - 速度
  - `bearing` - 方向（根据运动轨迹计算）
  - `accuracy` - 精度
  - `altitude` - 海拔

### 3. 轨迹运动更自然
- 添加了方位角（bearing）计算，根据移动方向自动更新
- 速度扰动算法保留（±15% + 正弦波）

## 使用方法

### 1. 授予权限
- 位置权限（必需）
- 通知权限（Android 13+）

### 2. 设置模拟位置应用
1. 打开 **设置 → 系统 → 开发者选项**
2. 找到 **"模拟位置信息应用"**（或 "选择模拟位置应用"）
3. 选择 **"虚拟跑步"**

### 3. 规划路线
1. 打开应用，等待地图加载
2. 点击地图选择 **起点**
3. 继续点击添加 **途经点**（可选）
4. 点击选择 **终点**

### 4. 开始虚拟跑步
1. 调整配速滑块（3-15 分钟/公里）
2. 点击 **"开始"** 按钮
3. 应用开始发送虚拟位置（1Hz 频率）

### 5. 验证定位是否生效
打开其他地图应用（如高德地图、百度地图），查看当前位置是否跟随虚拟路线移动。

## 注意事项

1. **Android 版本要求**：Android 8.0+ (API 26+)
2. **模拟位置设置**：必须在开发者选项中设置本应用为模拟位置应用，否则无法发送虚拟位置
3. **后台运行**：开始跑步后，服务会在前台运行，显示通知
4. **定位精度**：虚拟定位精度设置为 3-5 米，模拟真实 GPS 精度

## 常见问题

### Q: 地图还是显示白屏？
A: 请检查：
1. 手机是否联网（首次加载需要下载地图样式）
2. 是否授予存储权限（用于地图缓存）

### Q: 虚拟定位还是不生效？
A: 请检查：
1. 开发者选项中是否正确设置本应用为模拟位置应用
2. 目标应用是否使用 FUSED_PROVIDER（本应用已适配）
3. 尝试重启目标应用

### Q: 为什么有些应用检测不到虚拟位置？
A: 某些应用（如微信、支付宝）有自己的位置验证机制，可能检测到模拟位置。这是正常的安全机制。
